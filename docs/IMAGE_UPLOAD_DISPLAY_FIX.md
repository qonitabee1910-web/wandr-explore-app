# Image Upload Display Fix

## 🐛 Problem Found & Fixed

### Masalah:
Upload gambar → image tidak muncul di canvas → error 400

### Root Cause:
`getPublicUrl()` method dari Supabase tidak generate URL dalam format yang benar.

**Incorrect URL (from getPublicUrl)**:
```
https://bettsililhodvxiuzdvr.supabase.co/storage/v1/object/seat-layouts/layout-uuid.png
                                                          ❌ Missing "/public/" part
```

**Correct Public URL (fixed)**:
```
https://bettsililhodvxiuzdvr.supabase.co/storage/v1/object/public/seat-layouts/layout-uuid.png
                                                          ✅ Has "/public/" part
```

---

## ✅ Solution Applied

### Changed: `src/admin/services/seatLayoutService.ts`

**Before**:
```typescript
// This generates wrong URL format
const { data: publicUrlData } = supabase.storage
  .from('seat-layouts')
  .getPublicUrl(filename);

const publicUrl = publicUrlData?.publicUrl; // ❌ Wrong format
```

**After**:
```typescript
// Manually construct correct public URL
const supabaseUrl = supabase.supabaseUrl;
const publicUrl = `${supabaseUrl}/storage/v1/object/public/seat-layouts/${filename}`;
// ✅ Correct format with /public/ path
```

## 💡 Key Change

The fix ensures that public URLs generated by Supabase are in the correct format:
```
https://{PROJECT}.supabase.co/storage/v1/object/public/{BUCKET}/{FILE}
                                               ↑↑↑
                                        Must have "/public/"
```

This allows browsers to access public storage files directly without 400 errors.
